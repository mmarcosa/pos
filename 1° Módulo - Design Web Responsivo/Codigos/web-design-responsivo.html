Aula 4.03

Janela de visualização (viewport) é a área da tela destinada a apresentação de conteúdo.

Nos computadores, a janela de visualização corresponde a área de conteúdo do navegador

Nos smartphones, a janela de visualização pode ser maior ou menor que a janela do navegador.

Exemplo de ocmo definir a largura para navegadores de smartphones: 

                    < meta name = "viewport" content = "device-width">

Atributos de viewport:

width == define a largura do viewport

height == define a altura do viewport

initial-scale == define a escala inicial, cujo valor default é 0

minimum-scale == define o menor valor para a escala, podendo ser entre 0.1 e 10.0

maximum-scale == define o maior valor para a escala, podendo ser entre 0.1 e 10.0

user-scalable == define se o usuário pode alterar a escala

Aula 4.04

Uma media query é uma expressão lógica que testa o tipo de mídia do agente do usuário e, opcionalmente, uma ou mais de suas características.

Permite que a apresxentação da página seja ajustada por meio de css.

            
            @media screen{
                p{
                    color:#99F;
                    background-color:#CCC;
                }
            }
            
            
            @media print{
                p{
                    color:#000;
                    background-color:#FFF;
                }
            }

Parâmetros de uma media query

width - indica a largura da janela de visualização

height - indica a altura da janela de visualização

device-width - indica a largura da janela do dispositivo

device-height - indica a altura da janela do dispositivo

orientation - indica de a navegação é no modo retrato ou paisagem

aspect-ratio - indica a razão entre altura e largura da janela

monochrome - indica se o dispositivo é monocromárico

resolution - indica a resolução da tela (em dpi ou dpcm)

Os prefixos min- e max- permitem especificar faixas de valores. Exemplos:

min-width largura mínima

max-width largura máxima

                @media all and (min-width: 600px)
                
                @media handheld and (max-width: 240px)
                
                @media only and (orientation: landscape)

Exemplos de declarações para diferentes tamanhos:

            @media only all and (min-width: 320px){
                //conjunto de estilo
            }
            @media only all and (min-width: 640px){
                //conjunto de estilo
            }

No exemplo acima, as regras da próxima faixa incorporam as regras da faixa anterior.

Aula 4.05

Web única

A mesma informação e os mesmos serviços devem estar disponíveis aos usuários independentemente do dispositivo de acesso que eles usassem, pelo menos enquanto isso fizer sentido.

Isso não significa que exatamente a mesma informação e a mesma representação fossem usadas.

Layouts fluídos: a apresentação é ajustada a partir da largura da janela de visualização.

Melhorias progressivas: o layout é elaborado para dispositivos mais simples e melhorado para dispositivos mais complexos.

Layout fluído: as dimensões dos elementos são calculadas a partir das dimensões da janela de visualização

Layout adaptativo: os elementos são determinados a partir dos recursos e das características dos dispositivos

Técnicas de layout adaptativo

Detecção de recurso: teste no navegador, de quais recursos estão disponíveis. Recurso: Modernizr

Detecção de dispositivos: identificação de qual dispositivo (e navegador) está sendo usado pelo usuário. Recurso: User Agent String

Nada impede que essas técnicas sejam combinadas.

Aula 4.06

Modernizr

Um dos principais recursos para desenvolvimento responsivo. É uma pquena biblioteca em javascript para detectar quais recuross da HTML5 e da CSS3 estão disponíveis no navegador. http://modernizr.com

A inclusão deve ser feita como qualquer outro arquivo .js

Também permite a inclusão de uma nova classe para cada recurso testado.

Polyfills: scripts que implementam recursos não disponíveis de forma nativa no navegador Sua carga não deve ser feita no cabeçalho, e sim através do método Modernizr.load{..}

Aula 4.07

User Agent String

A User Agent String é a assinatura do agente do usuário (ex.: navegador)

Cada versão, em cada dispositivo, de cada agente do usuário possui uma User Agent String diferente

Essa assinatura faz parte do protocolo HTTP

Exemplod de ahente de usuário:

  • Navegadores
  • Programas de email
  • Web Services
  • Agregadores RSS
  • Máquna de busca

Veja uma extensa lista em: http://useragentstring.com/

Estratégias de detecção:

  • 1. Pesquisar por termmos na User Agent String (mobile, mobi, iphone etc)
  • 2. Consultar um Device Description Repository(WURFL)

Aula 4.09 - Bootstrap

Bootstrap é um framework responsivos. Nada mais é do que um conjunto de código em css, html e javascript que gerenciam layout e garantem que as páginas serão automaticamente adaptadas aos diferentes contextos de navegação.

Bootstrap necessita do jquery para funcionar.

Pode ser usado localmente, via get bootstrap ou via CDN.

Tanto o jquery quanto o bootstrap são colocados apenas no final da página, para evitar que a página demore a carregar.

Respostas:

  • .fail() --> Requisição malsucedida
  • .always() --> sempre
